# Changelog — 2026-07

### 2026-07-02 — fix(box/cachyos): generalize the envd-guest-home $HOME-bug check beyond one operator's literal username

The `envd-guest-home` check in `check-cachyos-gpu-vm` (proving the cross-host npm
builder's deploy-time `{{.Home}}` env.d resolution didn't leak the HOST operator's
home directory into the guest) hardcoded `grep -q "/home/atrawog"` — a real
regression-test weakness: it only ever caught the leak for one specific
developer's username, so the same bug would silently pass undetected on any other
operator's build host. Replaced with a dynamic check: extract every `/home/<user>`
path baked into the guest's `nodejs.env`, and assert each one equals the GUEST's
own actual `$HOME` at check-run time — any differing ("foreign") path fails the
check, regardless of what that foreign path happens to be. Edited both the `check:`
display field and the `plugin_input.command:` executed script identically (R3, no
duplication between the two).

Verified: `charly box validate` (exit 0). The shell logic itself was directly
proven live (not via the bed — see below) against synthetic fixtures reproducing
both a healthy guest env.d (only the guest's own home present — PASS) and the
$HOME bug with a foreign home leaked in (`/home/some-other-operator/...` — FAILS
with the new foreign-path message); confirmed the OLD hardcoded check would have
silently missed that exact same foreign-leak scenario, proving the fix was
genuinely needed. This candy's own R10 bed, **check-cachyos-gpu-vm, requires
nvidia GPU passthrough the build host lacks** (an AMD Phoenix APU, no NVIDIA
device present per `lspci` — not vfio-switchable, unlike the spice dep-shed
precedent above which hit the identical wall) — the bed's `vm-build` step
succeeded live via the real toolchain (proving the edited YAML parses and builds
correctly), but `vm-create` hard-errors at the resource-arbiter stage before the
guest ever boots, entirely unrelated to and unaffected by this change (it fails
identically on the unmodified file) — to be R10'd on a GPU host.

Assisted-by: Claude (analysed on a live system)
